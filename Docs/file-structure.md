# File Structure

## Current folder structure
```
personal website/
├── personal-site.html          (early copy in the root)
├── personal-style.css          (early copy in the root)
├── Codes/                      (the code folder)
│   ├── personal-site.html      (DESIGN 1 – main version: page)
│   ├── personal-style.css      (DESIGN 1 – main version: styles)
│   ├── code 2/
│   │   ├── personal-site-option2.html   (DESIGN 2: page)
│   │   ├── personal-style-option2.css   (DESIGN 2: styles)
│   │   └── Images - Copy/      (extra copy of images, not needed)
│   └── code 3/
│       ├── personal-site-option3.html
│       └── personal-style-option3.css
├── Images/                     (all the photos used by the website)
└── Docs/                       (this documentation folder)
```

## What each important file does
- **Codes/personal-site.html** – The main website page (Design 1). This is the one
  to show first.
- **Codes/personal-style.css** – The styles (colours, layout, fonts) for Design 1.
- **Codes/code 2/personal-site-option2.html** – The second design of the page.
- **Codes/code 2/personal-style-option2.css** – The styles for the second design.
- **Images/** – Holds all the photos (helmet, kart, driving, etc.).
- **Docs/** – Holds all the notes and documentation about the project.

## Code folder, Images folder and Docs folder
- **Code folder (`Codes`)** – all the HTML and CSS files live here.
- **Images folder (`Images`)** – all the pictures live here, at the project root.
- **Docs folder (`Docs`)** – all the written documentation lives here.

## Difference between the first design and the second design
- **Design 1 (main)** – `Codes/personal-site.html` + `personal-style.css`.
  A red, black and white racing look with image cards in the hero and a grid gallery.
- **Design 2 (option)** – `Codes/code 2/personal-site-option2.html` +
  `personal-style-option2.css`. The same content with a different layout, a stats
  band near the top and a different hero style.

Both designs use the **same images** from the root `Images` folder.
